new workflow

Scout a quantum topic with arXiv PDFs and quote evidence

--pipeline scout · live literature · bounded arXiv corpus · source KB · exact quote substantiation

The scout is the preferred broad discovery entry point. It writes a visible evidence trail: global_literature/, arxiv_corpus/, source_kb/, substantiation/, claim_ledger.md, scout_references.bib, and scout_report.md.

$ chain/run.sh --pipeline scout \
    --topic "Using AI for designing superconducting quantum chips" \
    --source-file seed_paper.pdf \
    --scout-arxiv-max-downloads 10 \
    --llm codex \
    --outdir runs/scout_demo

Offline smoke and acquisition-only modes are also available: add --no-scout-live-literature --no-scout-arxiv-corpus --no-llm, or use --scout-pdf-kb-only to build/search the corpus without idea generation.

Ground patent review in named prior art

patent_prior_art generates patent-search queries, searches keyless Google Patents/DDGS paths, filters on-topic records, and writes prior_art_refs.json for patent_reviewer --prior-art.

$ skills/patent_prior_art/run.sh \
    --topic "surface-code decoder using neural belief propagation" \
    --max 20 \
    --output runs/prior_art

$ skills/patent_reviewer/run.sh \
    --mode full \
    --patent US10614371B2 \
    --prior-art runs/prior_art/prior_art_refs.json

Chain workflow options

Both papers are routed through chain/run.sh --pipeline paper-audit. That preset names four default-on stages and two opt-in extras; toggle them with --skip-X / --with-X flags. 

$ bash QuantumNovelty/chain/run.sh --list-stages
# QN chain — stage table

pipeline          default-on stages                  optional
----------------------------------------------------------------
paper-audit       research, reviewer, fallacies, cqe novelty-audit, cross-llm
full              literature, discovery, audit, ...  -
mid-entry-stage-2.5  literature-1.5, audit, ...      -
mid-entry-stage-4 revision, re-review, cqe           -

Toggle defaults via per-stage flags:
  --skip-<stage>     turn off a default-on stage
  --with-<stage>     turn on an opt-in stage
  --pause-after S    write checkpoint after stage S
  --resume-from S    treat earlier stages as complete
  --force            re-run completed stages

Skip an unwanted stage: ./run_two_papers.sh --skip-fallacies
Run only the reviewer panel: ./run_two_papers.sh --skip-research --skip-fallacies --skip-cqe
Add cross-LLM falsifiability: ./run_two_papers.sh --with-cross-llm --hamiltonian H2 --geometry-sweep "R=0.7,0.9,1.1 A" --llms claude,codex

Grand total: $2.90 real USD across both papers (flowvqe $0.97 + lcutrotter $1.94) · run on the Claude Code CLI backend (the framework default) — exact token counts and per-call USD from the claude --output-format json envelope, no estimates. Models: claude-opus-4-5 / claude-haiku-4-5. Per-paper machine-actionable quality gates: flowvqe 6.0/10 major-revisions · lcutrotter 6.67/10 minor-revisions.

three-way head-to-head

QN vs ARS vs ARC — same paper, three pipelines

LCU-Trotter (arXiv:2212.04566, PRX Quantum)  ·  40-page comparison PDF

The same PRX Quantum paper, run end-to-end through three frameworks:

  • QuantumNovelty's paper-audit chain — 4 stages (research / reviewer / fallacies / cqe).
  • academic-research-skills' academic-paper-reviewer skill — 7-agent orchestration (field-analyst → EIC + 4 reviewers + Devil's Advocate → editorial synthesizer).
  • AutoResearchClaw's review stages — peer_review (Reviewer A + B markdown) + quality_gate (numeric verdict JSON with threshold gate). 2-stage subset of ARC's full 23-stage pipeline.

QN paper-audit output

4 stages, full prose:
research · reviewer panel (5 voices in one) · fallacies · CQE

Stage telemetry: pipeline_summary.json · HEARTBEAT_AUDIT.md

ARS output (7 agents)

field-analyst · EIC · methodology · domain · perspective · Devil's Advocate · editorial decision

ARC output (2 stages)

peer_review (Reviewer A + B markdown) · quality_gate (verdict, score 7/10)

ARC verdict (claude run): "Accept with minor revisions. The paper presents a novel and technically sound approach…"

The comparison PDF includes a 3-column coverage matrix: the 5-voice peer review pattern is shared QN↔ARS; QN adds quantum-CS-specific layers (audit-and-falsify checklist, 11 quantum-CS fallacies, Stage-6 CQE); ARS adds dedicated field-analyst + editorial-synthesizer agents; ARC adds methodology-evidence consistency check + numeric quality threshold gate (the only framework producing a numeric pass/revise verdict bound to a threshold).
